Block & Inline Elements

Hidden Properties

You may have already noticed: HTML elements take up space.

Each element has a hidden property. It is either a block or an inline element.

Block Elements

  • take up the entire width available to them
  • begin on and create new lines
  • may contain other block or inline elements

Elements with Block Properties

  • h1, h2, h3, h4, h5, h6
  • p
  • ul, ol and li
  • blockquote
  • HTML5 structural tags

Inline Elements

  • only take up the space within its surrounding tags
  • do not begin on or create lines
  • may only contain other inline elements

Elements with Inline Properties

  • a
  • abbr
  • del, ins
  • em, strong
  • mark

Elephants & Snakes

Block elements should not go inside inline elements...

... just like elephants do not belong inside snakes.

Block or Inline?

Chrome Developer Tools can help indicate an element's display property.

By hovering the mouse cursor over a line of code (or using the "select an element" setting), a visual cue will appear in the body to show whether the element takes up 100 percent of the space available to it (block), or just the space it needs (inline).

Another option: In the right panel of the "Elements" console of Chrome Developer Tools, look at the display value in the user agent stylesheet.

div

A div tag is a block element which does not have any semantic meaning.

W3C:

Authors are strongly encouraged to view the div element as an element of last resort, for when no other element is suitable.

span

A span tag is an inline element which does not have any semantic meaning.

W3C:

The span element doesn't mean anything on its own, but can be useful when used together with the global attributes, e.g. class, lang, or dir.

Tip

Always use semantic tags before resorting to a span or div tag.

Only use span and div when presentational grouping is required and no semantic tag will suffice.
