HTML Best Practices

Develop good habits

Understanding how to code is one thing; coding properly is another.

Developing good habits now will help you avoid bugs as you are learning and when you tackle more complex problems.

Always use the following best practices when coding HTML.

Name files properly

Name all files for your web site using lowercase letters and no spaces. Avoid special characters.

Hyphens (-) or underscores (_) are acceptable in lieu of spaces, though hyphens are better for SEO purposes.

Bad: Quorra's Home Page.html

Good: quorra-home-page.html

Some computer may not see photo.jpg and photo.JPG as being different, but case-sensitive web servers sees these as different files.

Using spaces or special characters in file names introduces hidden entities. A file named miami beach.jpg on your computer will be seen as miami%20beach.jpg on a web server.

Write code in lowercase

Write all HTML elements, attributes and values in lowercase.

Bad: <P CLASS="Intro">This is a sentence</P>

Good: <p class="intro">This is a sentence</p>

It is acceptable to use uppercase letters in alt or title values as these attributes contain content.

<img src="my-cat.jpg" alt="Flynn the Cat" />

Indent code

The use of whitespace to indent and separate lines aids readability.

Bad:

<ol><li>cat</li>
<li>dog</li>
<li>fish</li></ol>

Good:

<ol>
  <li>cat</li>
  <li>dog</li>
  <li>fish</li>
</ol>

Indent code by two spaces for each level. This makes it easier to 'match' opening and closing tags.

If you use the TAB key for indenting, make sure you set tab length in your code editor's preferences to two spaces.

Comment code

<!-- This is a comment. -->

Comments enable you (or whomever inherits your code) to:

  • identify parts of a page easily
  • understand the logic behind the code
  • explain what each part does

Comments and proper style will aid page maintenance.

Use a DOCTYPE

A DOCTYPE tells the web browser which version of HTML to use.

DOCTYPE we will use indicates our code is written in HTML5:

<!DOCTYPE html>

When DOCTYPE is missing, the browser renders the page in quirks mode, which may break a web page.

Declare a language

Setting the default language of a page can positively impact:

  • hyphenation
  • font selection
  • search
  • spell checkers
  • translation tools

Set a language by adding the lang attribute and a two-letter language code to the html element.

<html lang="en">

The lang attribute may be used with other tags if a document is multilingual.

Declare a character set

A character set indicates how letters, numbers and special characters should be displayed.

UTF-8 includes 1.1 million characters and is used by 85 percent of the web.

The following tag should appear in the <head> of a page:

<meta charset="UTF-8" />

Close tags

Bad: <p>example

Good: <p>example</p>

If an element is self-closing, add a slash (/) at the end of the tag.

<img src="my-cat.jpg" alt="Flynn the Cat" />

Provide attribute values

Most attributes require a value. A few do not.

Add a value, even when not required.

<audio controls="controls"></audio>

Quote attribute values

Unquoted attributes can lead to broken code.

Bad: <img src=photo.jpg alt=Sebastian the Ibis />

In the example above, a browser may mistake "the" and "Ibis" as attributes.

Good: <img src="photo.jpg" alt="Sebastian the Ibis" />

Use double quotations (""), not single quotations ('').

Encode hidden HTML entities

HTML entities (or 'escape codes') are special characters. They begin with an ampersand (&) and end with a semicolon (;).

Whenever you want to use less than (<), greater than (>) or ampersand in HTML, it is necessary to encode these.

If ever a foreign character or symbol typed into code appears broken, it may need to be encoded. Character Code provides an awesome reference for HTML entities. The W3C provides a comprehensive list, too.

Use valid HTML

Run your HTML documents through the W3C Validator to test.

Coding with valid HTML assures forward and backward compatibility and limits problems when implementing CSS and JavaScript. The validator can help find errors or bugs in code.

Do not use deprecated elements

Refrain from using tags which are no longer supported by the W3C specifications.

Separate presentation from structure

HTML has tags which are or have been used for visual presentation. These include <b>, <i> and <hr/>.

The presentational functionality of these tags should be handled via CSS.

Make sure your HTML focuses solely on structure.
