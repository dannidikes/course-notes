What is a server?

A computer set up to handle a service requests from clients on a network.

  • database
  • file
  • e-mail
  • print
  • web

A server can be a single, physical machine which handles multiple tasks. Most servers — for speed and efficiency — handle a single task.

Application Protocols

Computers and servers connect through transfer protocols (i.e. TCP/IP).

The applications which connect over a network have their own protocols.

  • FTP: File Transfer Protocol
  • NNTP: Network News Transfer Protocol
  • SSH: Secure Shell
  • SMTP: Simple Mail Transfer Protocol

Application protocols require TCP/IP in order to work.

HTTP

The web has its own protocol: Hypertext Transfer Protocol.

  • facilitates communication between clients and servers
  • a client (browser) requests a file from a web server
  • the web server acknowledges the request by sending a response
  • if the response is "OK!", the file requested is sent

Identifying Sites & Pages

How do we indicate which web site or file we are looking for on a server?

A client asks for a URL, or Uniform Resource Locator, a standard way of identifying a server, directory or file on the web.

A URL's structure consists of:

  • scheme name (protocol used to transfer the file)
    • http:
    • https:
  • hierarchy (the path to the file)
    • //com.miami.edu/about/about-the-school

Requests

The most common requests a client can make to a server are GET and POST.

  • GET retrieves data
    • most common request when browsing the web
  • POST sends data back to a server to be saved
    • post to a message board, update a database, submit a form

HTTP Status Codes

When a client makes a request, it will respond with a status code.

  • 1xx: Informational
  • 2xx: Success
    • 200: OK! (file transfer begins)
  • 3xx: Redirection
    • 301: Moved Permanently
  • 4xx: Client Error
    • 404: File Not Found
  • 5xx: Server Error
    • 500: Internal Server Error
