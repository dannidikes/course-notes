CSS Primer

What is CSS?

CSS stands for Cascading Style Sheets.

It is a language used to describe the presentation of markup.

Why "Cascading"?

When CSS is applied to an element, its children inherit the properties we establish.

For instance, if the following styles are applied to body:

  • font-size: 16 pixels
  • font-family: "Helvetica"

All text, regarless of surrounding tags (i.e. p, li, blockquote, etc.) be displayed in Helvetica, 16 pixels.

Presenting Media

CSS is predominantly used for visual styling, however it may be used to present markup for:

  • print
  • handheld devices
  • television
  • projectors
  • speech
  • braille

CSS Evolution

  • Emerged from a desire to improve visual presentation
  • 1994: First draft published by Håkon Wium Lee
  • 1996: CSS 1 adopted by Netscape, Internet Explorer
    • limited to colors, fonts and text and 'box' properties
  • 1998: CSS 2.1 introduced in 1998
    • added new media types, positioning, user interface
  • Today: CSS 3
    • added animation, transitions, web fonts

Why use CSS?

  • design consistency
  • avoids duplication
  • allows for site-wide changes in one place
  • faster page downloads

A scenario

You maintain a web site of 100,000 pages. In every HTML page, you include:

  • presentational elements such as <b> or <hr/>
  • inline styles which set every <p> to display in Helvetica and lime green

Yikes. Stano is weeping.

Your boss

That looks awful. Get rid of the horizontal rules. Why are we randomly making things bold? Change the font and color. 😠

There goes your weekend

You now have to search through 100,000 pages, remove each <b> and <hr/> tag and change the font and color of every paragraph.

Will you find every instance?

Is it possible deleting tags may break the site?

What if, after all the changes, the boss wants something different?

A Better Option

Instead of hard-coding the look and feel of our site in HTML, we can add a style sheet to each page.

<link href="style.css" rel="stylesheet" />

Within this file, rules on how to style elements, classes or ids (or a combination of) are defined.

p {
  color: black;
  font-family: "Avenir", sans-serif;
}

Applying CSS to HTML

There are three ways to add CSS to HTML:

  • inline styles
  • the <style> tag
  • external style sheet

ALWAYS use an external style sheet.

Inline Styles

style is a global attribute.

Adding this attribute to a tag results in presentation becoming hard-coded in HTML. Its use contradicts the benefits of CSS.

NEVER use inline styles.

the <style> tag

Styles can be defined within the <head> of a document by including the style tag.

ONLY use the style tag for prototyping.

When the styles for a design are fine-tuned, extract them and place them in an external style sheet. Link all pages to the external style sheet.

NEVER build and/or publish site which uses a <style> tag in HTML. Its use contradicts the benefits of CSS.

External Style Sheets

Store CSS rules in a single file.

Within each HTML file, include a link in the <head> tag to this central CSS file.

<link href="css/style.css" rel="stylesheet" />

When rules are applied or changed in the CSS file, they will apply to all HTML documents which link to it.

Don't forget HTML

When styling a web page, ALWAYS make sure you are working with VALID HTML.

Open, broken tags or incorrect attribute values will break your CSS.

Build the HTML first. Then apply CSS.

If you adjust HTML as you style, re-validate the code after a change.
